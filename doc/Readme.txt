===============================================================================

　UnrealRemoco

===============================================================================


■ソフトの概要

　コンピュータをリモート操作する（リモートデスクトップやＶＮＣみたいな）ソフトです。
　見て呉れはＶＮＣに似ていますが、独自のプロトコルを利用しているのでＶＮＣではありません。

　★サーバーは画面をキャプチャしつつ SendMessage を投げ続けるアプリケーションです。
　　ユーザーを切り替えたり、画面をロックしたり、セキュリティで保護されたデスクトップに切り替わると、
　　クライアント側から操作不能になります。

　通信内容を暗号化できます。


■動作環境

　Windows 10
　確認していませんが Windows 7/8/8.1 でも動くかもしれません。

　.NET Framework 4 を使用します。


■インストール方法

　●クライアント

　　UnrealClient.zip の中身をローカルディスク上の任意の場所にコピーして下さい。

　●サーバー

　　UnrealServer.zip の中身をローカルディスク上の任意の場所にコピーして下さい。


■アンインストール方法

　レジストリなどは一切使っていません。
　ファイルを削除するだけでアンインストールできます。


■使い方

　各アーカイブ内の Readme.txt をご覧ください。


■チュートリアル

　●暗号化しないで使う

　　〜サーバー側〜

　　1. UnrealServer.zip を展開して、UnrealServer.exe を実行する。

　　2. TCPポート 55900 番を開放する。

　　〜クライアント側〜

　　1. UnrealClient.zip を展開して、UnrealClient.exe を実行する。

　　2. 上部メニューバーの 接続 / 接続 を選択する。

　　3. "ホスト名" にサーバーのホスト名又は IP アドレスを入力する。

　　4. "接続" ボタンを押す。


　●鍵ファイルを使って暗号化して使う

　　〜サーバー側〜

　　1. UnrealServer.zip を展開して、UnrealServer.exe を実行する。

　　2. タスクアイコンを右クリックして "設定" を選択する。

　　3. "暗号化なし" から "鍵を使って暗号化" に変更する。

　　4. "設定" ボタンを押す。

　　5. "生成" ボタンを押す。

　　6. "エクスポート" ボタンを押して「鍵ファイル」を保存する。

　　　6.1. この鍵ファイルを安全な方法でクライアントに持って行く。

　　7. "OK" ボタンを押して鍵の設定ダイアログを閉じる。

　　8. "OK" ボタンを押して設定ダイアログを閉じる。

　　9. TCPポート 55900 番を開放する。

　　〜クライアント側〜

　　1. UnrealClient.zip を展開して、UnrealClient.exe を実行する。

　　2. 上部メニューバーの 接続 / 接続 を選択する。

　　3. "ホスト名" にサーバーのホスト名又は IP アドレスを入力する。

　　4. "暗号化なし" から "鍵を使って暗号化" に変更する。

　　5. "設定" ボタンを押す。

　　6. "インポート" ボタンを押して、サーバー側で生成した「鍵ファイル」を読み込む。

　　7. "OK" ボタンを押して鍵の設定ダイアログを閉じる。

　　8. "接続" ボタンを押す。


　　★鍵ファイルはクライアント側で生成しても良いです。


　●パスフレーズを使って暗号化して使う

　　〜サーバー側〜

　　1. UnrealServer.zip を展開して、UnrealServer.exe を実行する。

　　2. タスクアイコンを右クリックして "設定" を選択する。

　　3. "暗号化なし" から "パスフレーズを使って暗号化" に変更する。

　　4. "パスフレーズ" を入力する。

　　5. "OK" ボタンを押して設定ダイアログを閉じる。

　　6. TCPポート 55900 番を開放する。

　　〜クライアント側〜

　　1. UnrealClient.zip を展開して、UnrealClient.exe を実行する。

　　2. 上部メニューバーの 接続 / 接続 を選択する。

　　3. "ホスト名" にサーバーのホスト名又は IP アドレスを入力する。

　　4. "暗号化なし" から "パスフレーズを使って暗号化" に変更する。

　　5. "パスフレーズ" を入力する。

　　6. "接続" ボタンを押す。


■注意

　サーバー側の「転送先ポート番号」は平文で通信可能なプロセスが使用します。
　インターネットや信頼できないネットワークに対してポートを公開しないで下さい。

　暗号化したサーバーは総当たり攻撃に耐性がありません。
　てなわけで、なるべくパスフレーズではなく鍵ファイルを使ってね。


■制限

　クライアントとサーバーを同じ環境で同時に実行することはできません。


■既知の問題点

　Ctrl+Alt+Delなど、クライアント環境も反応してしまう入力を何とかする。

　通常キーを長い時間押し続けていても、連打にならない。

　サーバー側の画面をロックしている間、操作できない。
　サーバー側で別のユーザーに切り替えている間、操作できない。
　セキュリティで保護されたデスクトップに切り替わっている間、操作できない。
　-- 対策として、UACを切り、スクリーンセーバーを無効、ログオフ・ユーザー切替を行わないようにする。。。
